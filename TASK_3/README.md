#DESCRIPTION

This Python script demonstrates a simple yet effective chatbot built using Natural Language Processing (NLP) libraries such as NLTK and spaCy. The chatbot is designed to interact with users in a conversational loop, recognize basic intents, and provide varied responses to make the interaction feel more natural.
The program begins by importing the necessary libraries and downloading NLTK’s tokenizer resources. It then loads the lightweight spaCy model en_core_web_sm, which is capable of performing entity recognition. A dictionary of responses is defined with three categories: greetings, farewells, and default replies. Each category contains ten different responses, giving the chatbot a total of thirty possible outputs. This variety ensures that the chatbot avoids repetitive answers and maintains user engagement.
The intent detection function uses NLTK’s word tokenization to break down user input into lowercase tokens. Based on simple keyword matching (e.g., “hi” or “hello” for greetings, “bye” for farewells), the chatbot determines the user’s intent. If no keywords are matched, the chatbot defaults to a generic response.
The main loop continuously prompts the user for input. For each message, spaCy processes the text to identify named entities. If a geographical entity (GPE) such as a city name is detected, the chatbot dynamically adjusts its reply to mention weather information for that location. This demonstrates how entity recognition can be integrated into rule‑based systems to make responses context‑aware.
The chatbot runs until the user types “bye,” at which point it selects a farewell response and exits. Overall, this script highlights the fundamentals of combining NLTK for tokenization and intent detection with spaCy for entity recognition, while keeping the design simple and lightweight. It is an excellent internship‑level project that showcases practical NLP skills and provides a working conversational agent.
